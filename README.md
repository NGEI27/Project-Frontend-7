# Project: Frontend-7 /Modern Web Development & Architecture

## 🎯 პროექტის მიმოხილვა
ეს რეპოზიტორია წარმოადგენს ვებ-დეველოპმენტის კურსის ფარგლებში შექმნილ პროექტს, რომლის მიზანია თანამედროვე სტანდარტების შესაბამისი, ოპტიმიზებული და სკალირებადი ფრონტენდ აპლიკაციის შექმნა. პროექტის ფარგლებში განსაკუთრებული ყურადღება დაეთმო კოდის სისუფთავეს (Clean Code) და არქიტექტურულ მოწესრიგებას.

## 🛠 ტექნოლოგიური Stack & ხელსაწყოები
* **HTML5:** სემანტიკური მარკაპი SEO ოპტიმიზაციისა და ხელმისაწვდომობისთვის (Accessibility).
* **Sass (SCSS):** CSS-ის პრეპროცესორი, რომელიც გამოიყენება სტილების მოდულურობისთვის.
* **Git/GitHub:** ვერსიების მართვის სისტემა კოლაბორაციული მუშაობისთვის.
* **WSL (Ubuntu):** Linux-ზე დაფუძნებული გარემო პროფესიონალური დეველოპმენტისთვის. ახალი დავაყენე და გიტბაშის ნაცვლად გამოვიყენე:

## 🏗 არქიტექტურული მოდელი: Sass 7-1 Pattern
პროექტის სტრუქტურა  გაკეტდა **Sass 7-1** პრინციპის მიხედვიტ , რაც გულისხმობს სტილების გადანაწილებას 7 სხვადასხვა საქაღალდეში:

1. **Abstracts:** ცვლადები (`$colors`, `$fonts`), Mixins და Functions.
2. **Base:** საბაზისო სტილები, Reset და Typography.
3. **Components:** დამოუკიდებელი ელემენტები (Buttons, Inputs, Modals).
4. **Layout:** საიტის ჩონჩხი (Navigation, Footer, Grid System).
5. **Pages:** კონკრეტული გვერდების უნიკალური სტილები.
6. **Themes:** მუქი/ღია თემების მხარდაჭერა (საჭიროებისამებრ).
7. **Vendors:** გარე ბიბლიოთეკების სტილების გადაფარვა.

## 🔐 უსაფრთხოება და გარემოს კონფიგურაცია
პროექტის გამართვისას შესრულდა შემდეგი ტექნიკური დავალებები:
* **SSH Key Authentication:** გამოყენებულია **ED25519** ალგორითმი GitHub-თან უსაფრთხო კომუნიკაციისთვის.
* **Linux Permissions:** მოწესრიგდა ფაილური სისტემის უფლებები (`chown`) WSL გარემოში გამართული მუშაობისთვის.
* **Git Flow:** დანერგილია **Branching** სტრატეგია, სადაც ყოველი ახალი ფუნქციონალი (Feature) მუშავდება იზოლირებულ შტოში.
